🚀 远程 OpenClaw HTTPS 访问配置总结

问题

远程服务器 49.232.157.149 的 OpenClaw Control UI 通过 HTTPS 访问时显示 "pairing required"

解决方法

1. 修复 Gateway 配置

原来 auth.mode: open 不被允许，改成 token 模式：

"auth": {
    "mode": "token",
    "token": "test123"
}

2. 修复 Nginx 配置

WebSocket 代理需要特殊配置，添加：

proxy_set_header Connection "upgrade";

3. 设备配对

批准设备连接：

openclaw devices list
openclaw devices approve <requestId>

4. 重启服务

Nginx 和 Gateway 都重启确保配置生效：

nginx -s reload
openclaw gateway restart

✅ 最终效果

  • HTTPS 访问地址： https://www.hackernel.com/?token=test123
  • 状态显示： 健康状况正常
  • 已配对设备： 可以正常连接
  • 域名： www.hackernel.com
  • Token： test123

📝 补充说明

  • 使用 Let's Encrypt 自动申请 SSL 证书
  • 证书会自动续期
  • Nginx 反向代理到本地 20000 端口
